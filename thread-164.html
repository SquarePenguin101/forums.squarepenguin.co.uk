Integration Into Tretflix

23-06-2014, 08:45 AM
Hi Guys

Thanks for the great project, makes my viewing of BBC content much easier as I have a bit of a bandwidth issue where I am.

Question for someone who is smarter than me :-). I running a VM by the name of Tretflix on a server on my network. It is an Ubuntu VM which integrates SABNZBD, Sick Beard, Couch Potato, NZBDrone, Headphones, Plexmedia center & Transmission using a very nice CLI.

http://tretflix.com/

The apps are all accessed using my web browser accessing the servers VM's Ip address.

I am trying to integrate get_iplayer with the web_pvr onto the server but am having a problem accessing the web_pvr interface from my pc.

I have installed get_iplayer onto the box, following the linux full install guide and am running the Web PVR manager using the "get_iplayer_web_pvr" and it comes up with the message to access the interface using the "http://127.0.0.1:1935" address. Obviously this is a local address so I am trying to access it at the server address:1935 which it is not able to do.

The server has no GUI installed so am not able to run a browser on it locally. If possible I would like to set up the Web PVR interface to run it remotely from my machine. Am I able to do this or do I have to access it locally? Could run it from the CLI using PUTTY but would be much easier just to run it from my browser on my own machine like I do with all the other apps in Tretflix.

Thanks

Ok, got it working, found a similar question on http://lists.infradead.org/pipermail/get...00907.html

Had to edit the config file at /etc/default/get_iplayer_web_pvr and change the listen command to 0.0.0.0:1935
