hlshd not included in default 'best' shortcut

user-30

Why is hlshd not included as a mode in the default best shortcut? Indeed, none of the hls modes now appear in the tvbest shortcuts. If memory serves, they were backup modes to hvf when introduced in 2.96?

Some more detail...

The default 'tvbest' shortcut is currently set to 25fps streams (i.e. without the --fps50 flag) and contains the following modes.

hvfxsd,dvfxsd,hlsvhigh,hvfxhigh,dvfxhigh,hlsstd,hlsxstd,hvflow,dvflow

The wiki table here states that the max resolution available by specifying --tvmodes=best without the --fps50 flag is 1280*720.

But following the details in the table here that's wrong, the highest resolution available from the modes contained in the 'tvbest' shortcut at 25fps is 960*540.

So, is hlshd supposed to be included in the tvbest shortcut as a back up still? If no (that would seem the case as they have their own 'hlsbest' shortcut), then the wiki table needs adjusting to indicate the correct highest resolution or the need to specify the hlshd/hlsbest mode/shortcut to get that resolution.
