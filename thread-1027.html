01-09-2016, 08:06 PM
You're comparing apples and oranges to some extent. The default TV streams (--tvmode=best) have twice the bit rate of the pre-2.95 default streams (Flash/RTMP). This is documented in the release notes and elsewhere. The default stream format (HLS) has also changed. It is streamed in chunked segments via HTTP, so of course it will appear bursty, particularly if the media servers are heavily loaded or the connection is congested. This format change is also documented in the release notes and elsewhere. With the default HLS streams, get_iplayer can fill a 200Mb pipe if upstream supports it, but there are many things that can reduce download speed. In order to form an educated guess as to whether or not your upstream connection performs worse with HTTP downloads than it does with RTMP streaming, you need to test programmes in both formats with roughly the same bit rate and duration, at roughly the same time of day. Given sufficient bandwidth, I think most people find HTTP better, but your situation may be different. Stick with whatever works for you.
