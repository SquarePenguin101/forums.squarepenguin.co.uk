The file you posted does not contain embedded artwork.  VLC or WMP will pick up a folder.jpg or AlbumArtSmall.jpg file (may be hidden files) in the same directory and display it as cover art.  I have no idea how such files would have beeen created, particularly in a clean directory as I directed for your last test run.   Where Mp3tag or Windows Explorer pick up the artwork from I can't say, but it isn't from an image embedded in your file.  Mp3tag can store downloaded artwork in folder.jpg, but I don't think it creates it by default.  Maybe you have some kind of Explorer extension installed on your machine.

Whatever the case, I'm bowing out.  Your log output and the associated M4A file prove that get_iplayer is working correctly.  It is not adding embedded artwork to output files when --no-artwork is specified.  Maybe someone else here will recognise your symptoms and can identify the culprit, but you've demonstrated to my satisfaction that it is not get_iplayer.
