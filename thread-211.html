This is a known bug in 2.86. The system options file is incorrectly decoded, so non-ASCII characters in the configured output path (like the Ã in Ãdouard) cause a problem. This is fixed in 2.87, which will be released later this month. Until then, override the output directory in the systems options file by creating a value in your user options file that contains only ASCII characters:

Code:
get_iplayer --prefs-add --output=C:\some\ascii\path



After you install 2.87, remove the temporary setting:

Code:
get_iplayer --prefs-del --output=C:\some\ascii\path
