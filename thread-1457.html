Won't load in Windows 10 due to broken PATH system environment variable

user-1877

10-07-2017, 11:07 PM
Good evening,

I am trying to run get_iplayer on my Windows 10 Pro machine (upgraded from Windows 7 pro.

I have un-instaled the old versioin I had, installed the latest, un-installed it and again re-installed it.

get_iplayer will not start - this is the message I get in the command prompt window:

Files\Java\jre1.8.0_111\bin";C:\WINDOWS\system32;C:\WINDOWS;C:\WINDOWS\System32\Wbem;C:\WINDOWS\System32\WindowsPowerShell\v1.0\;C:\PROGRA~1\ULTRAE~1;C:\Program Files\QuickTime\QTSystem\;C:\Program Files\Skype\Phone\;C:\Program Files\get_iplayer;C:\Users\cassidy\AppData\Local\Microsoft\WindowsApps;" was unexpected at this time.

I have tried turning the firewall of but it doesn't make any difference.

Anyone got any ideas?

Thanking you

user-2

10-07-2017, 11:33 PM
The firewall has nothing to do with it. Your PATH system environment variable has been messed up by other installers and is breaking the get_iplayer batch file. Those quotes are mismatched (and unneeded) for starters, and there may be other problems. Google for info on how to edit PATH.

user-1877

11-07-2017, 10:05 PM
Dinky,

Thank you for spotting that - I have removed the quotes from the PATH variable and it is working again.

Much appreciated.
