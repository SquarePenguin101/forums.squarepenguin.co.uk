If I set recording modes to 'HVF' in the recording tab of the web pvr manager and click apply, then click record on a programme in the search results, then a new browser tab opens as normal and I get the following output;

get_iplayer Output:

INFO Trying to stream pid using type tv
INFO: pid found in cache
Matches:
2738: Olympic Ceremonies: 2016 - Opening Ceremony, BBC One, b07mzrxt

INFO: 1 Matching Programmes

Recording complete

If in the Get_iPlayer command window I enter the command 'get_iplayer get --pid b07mzrxt --tvmode=hvf' then the download starts as expected. This is on windows 7 64 bit using the initial release of GiP 2.95.

Is the web pvr manager behaviour incorrect?
