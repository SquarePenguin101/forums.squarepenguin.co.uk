1) You are running version 2.83 of get_iplayer and you need to revert the changes you made to SWF verification in order to circumvent the error code 256 failure under the last version of get_iplayer.

Reinstalling should take care of number 1, and might have an effect on number 2. Try it and report your findings. A few additions to SP's instructions:

1. When you uninstall get_iplayer you are offered the choice of deleting your preferences, etc. If you did not elect to do so, you will need to take an additional step to remove the now-unnecessary fix for "error code 256". See the "Reverting Your Changes" section in this document.

2. After uninstalling, delete the directory
Code:
C:\Program Files\get_iplayer
if it wasn't deleted by the uninstaller. It may not be crucial, but it ensures a clean slate for re-install.

3. If it still doesn't work after re-install, don't be tempted to monkey around with the installed files. Your original problem had nothing to do with the rtmpdump installation per se, and then you somehow wound up with no rtmpdump at all, so leave the installation directory alone while you test things.

4. After re-installing, do as SP instructed and run your download command with the
Code:
--verbose
option. Also send the output from the command below, which shows system preferences created by the installer:
Code:
get_iplayer --show-options


5. If you're having the same problem after re-install, open the get_iplayer command prompt (Get_iPlayer shortcut from the Start menu) and run:
Code:
perl -e "system('.\RTMPDump\rtmpdump.exe --help')"


Something on your system is interfering with the launch of rtmpdump.exe from within get_iplayer, and this test command attempts to mimic that environment.
