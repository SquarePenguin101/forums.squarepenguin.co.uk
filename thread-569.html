Possible to post-add metadata?

user-1077

28-10-2015, 02:08 PM
Due to lack of space on a locigal drive, get_iplayer was unable to add metadata to completed downloads.

A .flv video file is complete and I've found commands to download a .xml file and the thumbnail but can find nothing in the documentation to combine/complete the process into a .mp4 file.

Is this possible from within get_iplayer?  Or does one have to probe the dark-arts of AtomicParsley et al?

The alternative is to delete the .flv file and re-download with --force option but I was curious (aka lazy).

Ditto with a radio programme.

The occasional downside to downloading overnight due to peak-time data-cap.

user-2

28-10-2015, 03:39 PM
Change the extension of the file from .flv to .partial.mp4.flv (.partial.m4a.flv for radio), which may already be the case if original re-mux failed, then re-download with --force.  rtmpdump should detect that the partial file is in fact complete, then go on to remux to .mp4 and tag file as normal. The file must be in its original output directory.
