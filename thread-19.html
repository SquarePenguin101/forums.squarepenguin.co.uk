When ffmpeg re-muxes the downloaded programme to put it in a MP4 container, it apparently writes an incorrect header value for audio sample rate, and the PS3 is dumb enough to believe it. Most software players will use the actual audio data to determine playback rate. See these mailing list threads for a discussion of the problem and some possible solutions:

http://www.mail-archive.com/get_iplayer@...02221.html
http://www.mail-archive.com/get_iplayer@...00533.html
http://www.mail-archive.com/get_iplayer@...02001.html

I don't have a PS3, so I can't personally vouch for any of the suggested solutions, so use whatever works best for you. The easiest to implement is probably to replace your ffmpeg binary with an ancient version. In practice, that means you back up and replace this file:

Code:
C:\Program Files\get_iplayer\FFmpeg\ffmpeg-1.2-win32-static\bin\ffmpeg.exe

with the version of ffmpeg.exe included in this bundle:

http://sourceforge.net/projects/mplayer-...z/download
