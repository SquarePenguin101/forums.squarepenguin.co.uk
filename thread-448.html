This is a long-standing issue. If connectivity is poor or bandwidth insufficient, the server may drop the connection such that rtmpdump can't tell that an error occurred, so get_iplayer carries on thinking the download completed OK. Given that you appear to be a long way from the UK, you can't be too surprised to encounter connectivity problems from time to time, particularly if you are using an overloaded or throttled VPN. Then again, UK users can encounter the same issue. It's possible you're just always hitting overloaded servers or misconfigured media streams, but that seems unlikely and is impossible to diagnose anyway. Bottom line: There is nothing get_iplayer can do about it, and the only thing you can do about it is try to find a better network.

And FWIW, the rtmpdump download progress counter always stops at 99%.
