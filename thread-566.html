If you tried without a VPN and you are UK based then I don't know why you'd get low quality downloads.

I downloaded the file without problem using --modes=best. It appears to me to be a bit 'soft' and not exactly the highest quality but not worse than the SD version.

If you are receiving an unwatchable file I'm not sure why in particular that might be. Perhaps you are getting the file from a different CDN? You can 'exclude suppliers' (--exclude-supplier with options akamai, limelight, level3 or bidi) to rule out one troublesome CDN but that's a long shot and I wouldn't expect that to work.

I suppose you could post the output of --prefs-show here to just to see if you have something in your preferences that is set incorrectly.

NOTE - If you are attempting to download the same file again, you'll need to add the --force flag to the command to do so and make sure no .partial.flv files are hanging around in the download folder as that can interfere too.
