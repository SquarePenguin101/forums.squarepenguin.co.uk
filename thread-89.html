2.82 in Vista finds pid but download fails

user-291

23-12-2013, 02:10 PM
Hi, I am running version 2.82 in Vista and always download programs using the pid rather than searching for the title.

Get Iplayer is accepting my commands as valid, connecting to iplayer, but then failing to download.

I tried adding --tvmode=best on the end BUT this only partially downloads an FLV file and them the get iplayer window has scrolling text about aborted download and re-connecting...

Any ideas?

I see that 2.85 is only listed for Windows 7 & 8. Would it work on Vista?

Here is what I am typing, as an example -

get_iplayer --get --pidb03mp53s

Or the same with --tvmode=best at the end.

My Get iPlayer used to work fine, quickly pumping out mp4 files to my directory, without the need to specify a tvmode...

Thanks for any help/ideas

23-12-2013, 02:52 PM
Short answer: Upgrade immediately. 2.82 is obsolete and partially broken. 2.85 works on XP and Vista as well as Windows 7 and 8.

user-2

23-12-2013, 02:55 PM
If you still have problems with 2.85, add
Code:
--verbose
to your command and post the get_iplayer output as an attachment. There is no way to know what is going on from just a general description.
