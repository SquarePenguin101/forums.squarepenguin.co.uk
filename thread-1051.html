KB3185319 appears to be the update that causes the problem. It breaks IE favourites as well, so there are complaints about it around the web. KB3185319 is a cumulative security update for IE 11, so probably not a good idea to uninstall it, though if you do uninstall it this problem disappears. Favourites can be fixed by amending ACLs on the corresponding .url files, but that doesn't seem to work for pvr_manager.url, perhaps because it is in %PROGRAMFILES% - not sure. For now, when the warning dialog appears just click "Open" to launch web pvr managerÂ as normal.

The problem appears to be limited to .url files in special folders, so there is a hinky workaround for people who resent the extra mouse click. You can copy pvr_manager.url to a normal folder (e.g., %USERPROFILE%) and amend pvr_manager.cmd to launch the copy rather than the .url file in the installation directory.
