06-05-2017, 11:17 PM
Whatever this is, I can't reproduce it. Follow our instructions to provide a full report so we can see what is happening on your system. Don't test with web pvr - run a pvr search on the command line. If a pvr search runs from the command line, that would point to this problem being isolated to the web pvr.

A couple of other suggestions: First, upgrade to v3.01. 3.00 is no longer supported, though I doubt this is a version issue. Second, don't leave ".*" at the beginning of your search string. It is still a valid regular expression, but those metacharacters at the beginning are redundant. They also make for slightly awkward file names for pvr searches that begin with ".".
