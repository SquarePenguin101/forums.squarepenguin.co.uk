(02-09-2018, 10:13 PM)I have refreshfuture 1 in my options (which I don't recall putting there), but deleting it (--pref-del) doesn't touch it and trying to add a zero value via --pref-add fails.
If you don't use the correct option names, you can't expect it to work. If you can't figure it out, just delete the line in the options file.
(02-09-2018, 10:13 PM)I'm also seeing exactly the same recurring cache corruption issue - I reported it before (here) and we ended up with the rebuild solution. As reported above, that works, but the issue recurs after a while.
Users won't be able to tell these apart without some extra effort, but don't bother. Get rid of --refresh-future, rebuild your cache, and all will be well.
